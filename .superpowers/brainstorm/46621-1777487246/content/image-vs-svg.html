Image vs SVG — What it actually looks like

Hero background is #f5f7ff (light blue). Both approaches rendered in context.

Actual JPEG — blend modes

Image file
mix-blend-mode: multiply (best option)
  • Background mostly blends away via multiply on light bg
  • Zero illustration work — use the image as-is
  • Photorealistic detail, holographic iridescence stays
  • Blend behaviour varies across browsers and themes
  • Image is ~1MB — needs resizing to ~150px WebP (~6KB)
  • Left side of robot (white/silver) gets darkened by multiply — loses the iridescence
  • No glow effect — can't apply CSS drop-shadow cleanly over image
  • If user is on white bg override — background is visibly a rectangle

Polished SVG — brand-styled

Recommended
Proper SVG (not the wireframe draft from before)
  • Fully transparent background — zero blending hacks
  • CSS drop-shadow / glow works cleanly on vector paths
  • Animated scan-line pulse for free — no extra weight
  • Scales perfectly to any DPR without blur
  • Orange antenna + orange eye trace echoes the orb's orange service nodes — connected visual language
  • ~3KB inline vs ~150KB WebP image
  • Takes time to illustrate properly (this is still a draft)
  • Doesn't have the photorealistic iridescence of the reference photo
The core trade-off: Using the actual image gets you the photorealistic holographic iridescence from the reference, but you're fighting CSS blend modes and accepting an opaque-ish rectangle at the orb's centre — the drop-shadow glow effect that makes the current M-mark look good won't transfer to a JPEG. The SVG approach gives you a transparent, glowing, animated icon that integrates into the orb as if it belongs there — but the iridescence is simulated with gradients, not photorealistic.

Middle path: We could make the SVG significantly more polished — the scan-line is already there, and we can push the holographic gradient (the left-face shimmer) much further with layered SVG gradients + CSS filter animations. The mockup above is ~30% of what the final SVG would look like.

Scroll-driven · the second motion layer

Your anchors lean on scroll. Linear's sticky stacks, Stripe's pinned API explainer, Vercel's reveal cadence, supermemory's section transitions. Locked stack below — all open-source, all aligned with the one-shot rule (scroll plays the animation; it doesn't replay it).

signature
ASCII text rebuilds character-by-character as it enters viewport. Our distinctive scroll moment. Cap at 2 per page.
parallax.restraint
0.2× max
Single element per section. ≤ 0.2 movement multiplier. No background-image parallax. No multi-layer.
Scroll motion — forbidden
  • Scroll-jacking that overrides natural scroll velocity for storytelling.
  • Spring / bounce easing. We're precise, not playful.
  • 3D transforms past 5 deg rotation. No mouse-tilt cards.
  • Background-image parallax — fights the SVG-on-grid posture.
  • Animations longer than 1500 ms — past the deliberate ceiling.
  • Cursor-following blob backgrounds, gradient followers.

Imagery direction — locked
Allowed four sources, in this order of preference
  1. 01Type and structure. The default. Sections carry themselves on the grid, the eyebrows, the numerals, the negative space. If a section feels empty, that's a layout problem to solve — not a hole to fill with imagery.
  2. 02Code-rendered SVG. Live diagrams, sparklines, timelines, blueprint frames — drawn in the DOM, accessible, scalable, themable. Reserved for technical content, not decoration.
  3. 03ASCII art and ASCII video — our signature. Rendered from real source material (a video, a photograph, a 3D scene) into mp4/webp via an ASCII renderer. Hero ASCII loop, contact-CTA ASCII landscape, ASCII chrome chips. This is the visual move none of our anchors make — it's what differentiates us.
  4. 04Real photographs of real people. Founders, team, an office, a whiteboard. One consistent edit — LUT locked separately. No stock. No models. If it isn't of someone we employ, it doesn't ship.
Forbidden do not introduce
  • Raster isometric illustrations (the prior why-us/*.webp set — retired).
  • Stock photography of any kind. People at laptops. Handshakes. Server racks. Glowing brains. Hands on keyboards.
  • AI-generated imagery of any kind — illustration, photography, or video. No exceptions, even for backgrounds.
  • 3D rendered scenes — Blender hero compositions, glass-blob hero imagery, abstract 3D shapes.
  • Cartoon, hand-drawn, or character illustration. Even commissioned. Doesn't fit engineering-premium grammar.
  • Decorative gradients used as backdrops, ambient blobs, abstract "design" filler.
  • Lottie animations. Static SVG and code-rendered motion only.

13 — Rules

Do. Don't.
Log deviations.

Hard constraints first. Sections may deviate when a signature visual demands it — log the deviation at docs/superpowers/specs/<date>-section-<name>.md.

Do

  • Must reference tokens — alias or CSS var. Never raw hex / px.
  • Must document all seven states for every interactive primitive.
  • Must render mobile fallbacks server-side when content has SEO value.
  • Must honour prefers-reduced-motion: reduce.
  • Must apply the focus-visible ring on every interactive primitive.
  • Must keep tap targets ≥ 44 × 44 px on touch.
  • Should prefer borders over shadows for separation.
  • Should prefer CSS over JS where a media query suffices.

Don't

  • Must not introduce one-off spacing or typography exceptions.
  • Must not hardcode horizontal section padding in inline styles.
  • Must not use color.text.tertiary for body copy.
  • Must not add a second html { @apply font-sans } — it breaks Switzer.
  • Must not use display:none to hide content with SEO value.
  • Must not introduce new pillar-tinted UI without updating services-data.ts.
  • Must not ship infinite animations outside the trust-bar marquee + orb HUD blink.
  • Must not invert dark mode — dark surfaces are redesigned, not inverted.

Implementation rules
  • 01Class-based toggle. Apply .dark to <html>. All token overrides live under .dark { --color-bg: ...; }. No per-component dark logic — components consume the same tokens.
  • 02First-visit default. Read prefers-color-scheme on first load, set the class before paint to avoid the FOUC flash. Persist user toggle in localStorage.theme and prefer it over OS afterwards.
  • 03Toggle placement. Top-right of the nav, sibling to "Talk to us". 16 px sun/moon icon in a 36 px square. Same focus-ring rules as every other interactive primitive.
  • 04Transitions. color, background-color, border-color transition at duration.fast (250 ms) when the class flips. Nothing else animates — no layout shift, no fade-cascade.
  • 05Imagery on dark. ASCII renders need a darker re-render — the same WebP at 50% opacity on dark looks anemic. Ship two variants: landscape.webp + landscape.dark.webp, swap via picture media query.
  • 06Photographs on dark. Founder photos with white backgrounds will look like cutouts. Re-edit with dark backdrop or use a 1 px brand-blue border to hold the frame.
Dark mode — anti-patterns
  • Pure black #000 backgrounds — too sharp. We use #0a0a0a.
  • Pure white #FFF body text — too high contrast. We use #FAFAFA.
  • Brightness inversion (CSS filter: invert()) anywhere. Surfaces are redesigned, not flipped.
  • Glowing accent halos behind text or cards. The brand pops without help.
  • "Cyberpunk" neon edges, scan lines, CRT effects. Engineering-premium, not gamer-aesthetic.

# Product Studio — Keyword + AEO Map (7 leaves)

Set-level research for the re-cut Product Studio pillar. One **distinct** primary
head-term per leaf; cannibalization-checked across the set. Volume/difficulty are
**estimates** — validate against Ahrefs/SEMrush before locking titles. Difficulty
tiers: T1 < 45, T2 45–60, T3 > 60.

> Inputs: senior product-engineering studio, India-based / global delivery, equal
> Web3 + AI positioning. No staff-aug. Anchors: SaaS→Chillies/CropXcel,
> WebApp→GetSmart, Mobile→Breayz, UI/UX→Absolveme; Discovery/MVP/DevOps = honest
> capability (no named case).

## 1. Product Discovery — `product-discovery`

- **Primary:** `product discovery services` — Low/Med vol · Diff ~34 · **T1**
- **Secondary:** product discovery sprint · product validation services · discovery phase software development
- **Intent:** commercial-investigative (pre-build founders de-risking scope)
- **AEO questions:**
  - "What is product discovery?" → definition sentence
  - "How long does a product discovery phase take?" → direct number (1–2 weeks)
  - "What are the deliverables of a product discovery sprint?" → numbered list
  - "Product discovery vs MVP — what's the difference?" → comparison (table/2-col)
  - "Do I need product discovery before building an MVP?" → direct yes + why
- **Owns:** "discovery / validation / before building". MUST NOT target "mvp development".

## 2. MVP Development — `mvp-development`

## Cannibalization matrix (the risky pairs)

| Pair | Risk | Resolution |
|------|------|-----------|
| SaaS ↔ Web App | High — both "build a web thing" | SaaS owns *multi-tenant subscription product* + "b2b saas"; Web App owns *bespoke/custom single-purpose web app* + "custom web application". Never let Web App target "saas development". |
| SaaS ↔ (old) B2B Multi-Tenant | Was two pages, same proof | **Folded.** "multi-tenant / b2b saas" is now a SaaS secondary, not a page. |
| MVP ↔ SaaS / Web App | Med — MVP is often a SaaS/web build | MVP owns *first-version / zero-to-launch / cost-of-an-mvp*; do not target "saas development" or "web application development" as primary. Cross-link instead. |
| Discovery ↔ MVP | Med — adjacent lifecycle | Discovery owns *validation / before building*; MVP owns *building the first version*. The "Discovery vs MVP" AEO question lives on the **Discovery** page. |
| Web App ↔ Mobile | Low | Distinct platforms; keep "web" vs "mobile/cross-platform" language clean. |
| UI/UX ↔ DevOps | None | Different disciplines. |

**Net:** 7 distinct primary terms, no unresolved overlap. Each leaf has ≥5 AEO
questions with assigned answer formats for the FAQ/definition blocks.

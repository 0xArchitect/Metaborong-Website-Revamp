## Findings considered and rejected

(So nobody re-audits these.)

- **Geo-override XSS bypass** (`lib/geo.ts` block overrides): rejected —
  `block_overrides` is Zod-validated at the PATCH boundary
  (`lib/blog-schema.ts:174`) and the renderer escapes via React.
- **Missing .env example**: rejected — `.env.local.example` exists, annotated.
- **`secure` cookie flag tied to NODE_ENV**: rejected — on Vercel, NODE_ENV is
  always `production` in deployed envs and previews are HTTPS; local dev needs
  `secure: false`. By design.
- **"All 35 hero visuals bundled into every leaf page"**: rejected — misreads
  RSC chunking; the browser only loads chunks for rendered client components,
  and most visuals are server-rendered SVG/CSS. Revisit only with bundle-trace
  evidence.
- **`countDocuments` per blog render**: rejected for now — deliberate,
  documented threshold in `lib/posts.ts` ("switch to a counter past ~10k").
- **Hero video Safari autoplay race**: rejected — known, commented, has a
  user-activation retry. Polish only.
- **Refactor 35 visuals' shared animation boilerplate / split 1.8k-line content
  files**: rejected — working, frozen, data-only; refactor risk exceeds value.
- **Repo-wide Prettier introduction**: rejected — a full reformat on a shared
  branch conflicts with the owner's surgical-changes rule.
- **Slug-immutability check-then-act race in `lib/posts.ts` updatePost**:
  acknowledged-real but not planned — single-admin CMS, millisecond window;
  revisit if the CMS ever gets multiple writers.
- **Multipart MIME pre-check on admin image upload**: rejected — magic-byte
  sniffing in `lib/images.ts` is already the authoritative gate.
- **MCP switch-statement default-case hardening**: rejected — TypeScript
  discriminated-union exhaustiveness already guards it.
- **Lint debt (~35 set-state-in-effect errors)**: owner-deferred, tracked
  outside these plans; CI (plan 002) runs lint non-blocking until it's paid.
